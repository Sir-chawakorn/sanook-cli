# Contextual Note Rule

> Notes must explain why they exist, not just what they contain.

## Required Context

- Why this note was created.
- What decision, task, entity, or source it connects to.
- What would make the note stale or superseded.
- Where the next reader should go next.

## Routing Check

Before creating a new note:

1. Search for an existing canonical home.
2. Check [[Vault Structure Map]] for the right folder.
3. If uncertain, park in [[Shared/Memory-Inbox/memory-inbox]] with a short reason.
4. Promote only after the context is clear.
